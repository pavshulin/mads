MADS test task

Description

Create Swipe image slider like in examples:

As you can see from videos it should have two animation types. It can be whether "Slide" or "Fade in" animation.

Requirements

  • Write code in Object Oriented style. Create base class and extended with two subclassed, one for "Slide" and another for "Fade in" animation types.
  • Do not use jQuery or other general purpose frameworks.
  • Use native JS or self written function for DOM manipulation and Events handling.
  • You can use libraries for Class inheritance emulation (such as jsclass) or for module loading (such as requireJS). Or you can use something self written. Or you can not use it at all.
  • Use CSS3 transitions for Slide and Fade in animations.
  • Slider should handle both Mouse and Touch events. That means user should be able to change slides by swiping by mouse or by finger (on touch devices).
  • Primarily this slider supposed for mobile devices, but it also should work fine in desktop browser.
  • Browser support:
    • Last version of Google Chrome
    • Any mobile browser on smartphone (Android, iOS, Windows Phone). If you don't have a mobile browser you can emulate it using Google Chrome Developer tools.

Swipe slider configuration

Use this configurational object for swipe slider:

var config = {
	// arbitrary number of images
	images: [
		'http://example.com/image-1.jpg',
		'http://example.com/image-2.jpg',
		'http://example.com/image-3.jpg',
		'http://example.com/image-4.jpg'
	],
	// possible values: 'auto', 'manual', 'automanual'
	mode: 'auto',
	// arbitrary interger (miliseconds)
	swipeSpeed: 500,
	// arbitrary interger (miliseconds). This is used in 'auto' and 'automanual' modes
	swipeDelay: 3000
};

Explanation:

  • images — Array of images that will represent slides of Swipe slider. Possible number of images: from 2 to ∞.
  • mode — String
    • auto — slides will be changed automatically. User can not interact with slider.
    • manual — only user should be able to change slides by making swipe gesture by mouse or by finger (on touch devices).
    • automanual — both of two previous. Slides are changed automatically and user is able to change them manually.
  • swipeSpeed — the speed of animation.
  • swipeDelay — delay between slides change in automatical swipe mode. Ignored in 'manual' mode.
